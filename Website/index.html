About

How exactly did we manage to pull this off?
Team Members: Josh Dunn, Ahmed Ismail, Christopher Rossi, Ryan Sartin
In case you haven't yet done the math, an 8x8x8 LED cube consists of 512 indidual LEDs, all soldered together in a pattern that allows you to light any desired LED. It is immediately obvious that each LED cannot have it's own pin, since the mbed does not support such a number of digital output pins. To handle this problem, we connect each LED's VDD leg in a column to form 64 8-long LED columns. Even with 64 columns, we decided to use I2C with I/O Expanders in orders to address all 64 columns, since the mbed still does not support this number of digital out pins. That takes care of trying to light any single LED in a column. To select the exact layer belong to the LED we want to turn on, we send GND (logic 0) to that layer. In order to do so, the GND leg of each LED in a layer must be connected, and then each of the 8 layers will be connected to the output of a 3-to-8 decoder, which will then be controlled by some digital out pins on the mbed. Components

Here's what we used to build this LED cube:
  1. 3mm Diffused Blue LEDs x 512 count
  2. 1 HD74HC138P 3-to-8 decoder
    • Used to select/power a layer of the cube.
  3. 8-16 NPN BJTs/NFETs
    • Used to ensure the necessary current to each layer.
  4. 4 MCP23017 16-bit I/O Expanders (I2C)
    • Used for powering one of the 64 columns
  5. mbed microcontroller w/ an external 5v power supply
  6. Solder/Soldering iron
  7. Solid wire
  8. PATIENCE

Future Work

What we would do with more time on our hands..
Now that we have built the LED cube, the only stopping point is our imagination and creativity. To start, we would like to incorporate a "better" gamepad. What we currently have is a seperate mini breadboard with 6 pushbuttons, 4 for each cardinal direction and 2 for moving up and down the layers. To improve on this, we could use a 3-D Joystick in order to control the direction of the snake more naturally. There is also the option of adding a bluetooth receiver to the schematic and using a Wii-mote's accelerometer to control the direction of the snake.
Besides the control of the game, there's always software manipulation. Of course, there's no limit to the kinds of things you can do with an LED Cube (e.g. a rain effect , ripples, or of course, WOOPWOOP.) The next stop for us will probably be to mess with some rotations:
